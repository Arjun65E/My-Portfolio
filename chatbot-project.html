🤖 Building an Offline AI Chatbot

AM/NS India Internship Project · June 2026 — A fully local LLM-powered chatbot with no internet dependency.

Local LLM Offline / Air-gapped Python AM/NS India June 2026

Project Overview

Goal
Deploy a chatbot that answers employee queries using company documents, with zero internet connectivity required.
Environment
AM/NS India internal network · Air-gapped / offline constraint
Core Tech
Local LLM · Vector DB · Document embeddings · RAG pipeline
Outcome
Working prototype serving safety procedure Q&A from company PDFs — fully offline.

Project Pipeline

1

Document Ingestion

Load company documents (PDFs, Word files) from a local directory. Parse and clean raw text content.

PDF parsingText extraction
2

Chunking & Embedding

Split documents into overlapping text chunks. Generate vector embeddings locally using a sentence transformer model.

ChunkingSentence TransformersEmbeddings
3

Vector Store

Store all embeddings in a local vector database (FAISS or Chroma) for fast semantic similarity search.

FAISS / ChromaVector DBSimilarity Search
4

Query & Retrieval (RAG)

User query is embedded → nearest document chunks retrieved from vector store → passed as context to the LLM.

RAGRetrievalContext injection
5

Local LLM Inference

A quantized open-source LLM (e.g. LLaMA / Mistral via Ollama) generates the final answer using retrieved context — entirely on-device.

OllamaLLaMA / MistralGGUF quantization
6

Chat Interface

Simple web UI (or CLI) where employees can ask questions and receive grounded answers with source references.

Gradio / StreamlitCLI
